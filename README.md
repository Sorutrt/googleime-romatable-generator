# googleime-romatable-generator

Google 日本語入力のローマ字テーブル向けデータを生成する小さな Go CLI です。

`preferences/` にある物理キーから出力への対応表を読み込み、次の 2 形式で出力できます。

単打の `a` `k` `s` などのローマ字割り当てがあれば、`ka -> か` や `kya -> きゃ` のような複合かな行も自動生成します。`a/i/u/e/o` の単打は `あ/い/う/え/お` に変換して出力します。`yぁ` `yぇ` のようなフラグメントキーがあれば、`t + yぇ -> ちぇ` のような合成も生成します。`kk` `tt` などの二重子音からは促音つきの `っか` `った` なども生成します。

- Google 日本語入力向けの 3 列 TSV
- コメントを書ける YAML 中間形式

## TL;DR

リポジトリ直下で:

```powershell
go run . -input preferences\qwerty_to_other -format yaml
go run . -input preferences\qwerty_to_other -format google -output romatable.tsv
```

## What It Does

通常の `key -> output` だけでなく、`*q` のような記法でキーをレイヤーとして扱えます。
また、`key -> a` や `key -> k` のような単打ローマ字定義から、標準的なローマ字かなシーケンスを自動で補完します。

たとえば:

```text
q   *q
j   h
g   f
```

この場合:

- `q` は単打で文字を出さず、`next input = q` を立てる
- `j` は単打で `h`
- `g` は単打で `f`
- 母音キー単打は `あいうえお` に変換される
- `h + d` のような子音 + 母音の組み合わせは、対応するかな行に自動展開される

さらに `-input` と同じディレクトリに `layer_q` という名前のファイルを置いて:

```text
h   ←
j   ↓
g   …
```

を定義すると、`qh -> ←`、`qj -> ↓`、`qg -> …` のような複合入力ルールに自動展開されます。

## Input Files

### Mapping File

`-input` に渡すファイルです。1 行 2 列で書きます。

```text
physical_key output
```

例:

```text
q   *q
w   *w
e   u
r   *r
t   *t
```

ルール:

- 空行は無視されます
- `#` で始まる行はコメントとして無視されます
- `*q` のような値は「出力文字」ではなく「レイヤーキー」を意味します
- `*q` の `q` は既存の物理キーとして定義されている必要があります
- `a` `k` `s` など 1 文字の英小文字は「ローマ字キー」としても扱われ、複合かな行の自動生成に使われます

### Layer File

`-input` と同じディレクトリに `layer_<layer名>` という名前で置く任意ファイルです。  
`q   *q` があるなら `layer_q` を置くと自動で読み込まれます。

```text
relative_key output
```

例:

```text
h   ←
j   ↓
g   …
```

この場合は `qh -> ←`, `qj -> ↓`, `qg -> …` に展開されます。

ルール:

- 空行は無視されます
- `#` で始まる行はコメントとして無視されます
- 同じ `layer_<name>` は、そのレイヤーを参照するすべての物理キーに展開されます

### Sequence File

`-sequences` に渡す任意ファイルです。複合入力を追加したいときだけ使います。

```text
sequence output
```

例:

```text
qj  ↓
qg  …
```

ルール:

- 空行は無視されます
- `#` で始まる行はコメントとして無視されます
- 同じ入力シーケンスを複数回定義するとエラーになります

## Usage

YAML 中間形式を標準出力に出す:

```powershell
go run . -input preferences\qwerty_to_other -format yaml
```

`layer_q` を自動展開しつつ Google 日本語入力向け TSV をファイルに書く:

```powershell
go run . -input preferences\qwerty_to_other -format google -output romatable.tsv
```

追加の手書きシーケンスも合わせる:

```powershell
go run . -input preferences\qwerty_to_other -sequences sequences.txt -format google -output romatable.tsv
```

## Output Formats

### `-format google`

Google 日本語入力向けの 3 列 TSV を出します。

```text
input<TAB>output<TAB>next
```

例:

```text
q		q
j	h	
qj	↓	
```

- 単打ルールは `next` が空です
- レイヤーキーは `output` が空で `next` にレイヤー名が入ります

### `-format yaml`

編集や確認向けの中間形式です。コメントを追加できます。

```yaml
# Generated by googleime-romatable-generator
single_rules:
  - input: "e"
    output: "u"
layer_keys:
  - input: "q"
    next: "q"
sequence_rules:
  - input: "qh"
    output: "←"
  - input: "qj"
    output: "↓"
```

## Validation

次の場合はエラーになります。

- 物理キーが重複している
- `*x` の参照先キーが未定義
- `layer_<name>` の内容が不正
- 入力シーケンスが重複している
- 単打ルールと複合ルールが同じ入力を奪い合う
